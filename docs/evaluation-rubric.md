# Evaluation Rubric

This rubric evaluates candidate review quality, not trading performance.

| Criterion                 | Description                                                      | Strong Evidence                                                                         |
| ------------------------- | ---------------------------------------------------------------- | --------------------------------------------------------------------------------------- |
| Context completeness      | The review note explains enough context for human review.        | Candidate summary, source context, timing concern, and missing information are visible. |
| Risk visibility           | Risk factors are explicit before any decision state is assigned. | Risk gates and review risks are listed clearly.                                         |
| Uncertainty disclosure    | Unknowns and weak assumptions are not hidden.                    | Missing information and uncertainty are documented.                                     |
| Human review readiness    | The note supports human judgment rather than automation.         | Review questions and decision states are separated from AI-generated text.              |
| Decision traceability     | The review decision can be understood later.                     | Human decision state, rationale, and follow-up state are recorded.                      |
| Post-event learning value | The item can support later audit and checklist improvement.      | Post-event review placeholder and write-back notes are included.                        |
| Public-safety compliance  | The item avoids sensitive or private trading information.        | No real signals, accounts, PnL, strategy rules, private prompts, or credentials appear. |
| Overclaiming avoidance    | The review does not imply profitability or automated execution.  | The language stays focused on review quality and human judgment.                        |

## Scoring Approach

Each criterion can be scored from 1 to 5:

- 1: Weak or missing
- 2: Present but inconsistent
- 3: Functional baseline
- 4: Strong and repeatable
- 5: Mature and adaptable

## Portfolio Interpretation

For this public repository, the rubric demonstrates how candidate review quality
could be evaluated. It does not evaluate financial returns, trading performance,
or strategy profitability.
