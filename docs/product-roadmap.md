# Product Roadmap

## Stage 0: Public-Safe Portfolio Documentation

Current stage.

Goals:

- Explain the concept safely
- Document review workflow and risk gate logic
- Add synthetic review note example
- Define responsible AI and public-safety boundaries
- Avoid real signals, account data, and strategy rules

## Stage 1: Candidate Review Template

Goals:

- Refine candidate review fields
- Improve missing-information prompts
- Standardize human decision states
- Keep the template separate from trading execution

## Stage 2: Risk Gate and Decision Log Structure

Goals:

- Define conceptual risk gate categories
- Create a public-safe decision log format
- Add review rationale fields
- Avoid proprietary thresholds or filters

## Stage 3: Quality Audit and Post-Event Review Loop

Goals:

- Add post-event review template
- Define review quality scoring
- Track false positives and late candidates conceptually
- Improve checklist updates from lessons learned

## Stage 4: Public-Safe Dashboard Mockups and Synthetic Examples

Goals:

- Create synthetic dashboard data
- Build public-safe review note screenshots
- Add quality audit mockups
- Prepare screenshots only after all four portfolio repositories are complete

## Stage 5: Enterprise-Style Risk Review Workflow Adaptation

Goals:

- Adapt the workflow for enterprise review and quality assurance
- Explore risk gate review outside trading contexts
- Add role-based review concepts
- Connect the system to broader auditability and governance workflows

## Roadmap Principle

The roadmap is intentionally staged. AI Quant Review System should be presented
as an AI-assisted review workflow and risk-aware decision-support concept, not
as an automated trading platform.
